# Sidebar.tsx 코드 리뷰

## 발견한 문제
- 스크롤 이벤트마다 `lastScrollY` 상태가 갱신되어 `useEffect`가 매번 재실행되고, 그때마다 스크롤 리스너를 제거했다가 다시 등록함. 모바일에서 빠르게 스크롤하면 성능 저하 위험이 있었음.

## 개선 내용
- `lastScrollY`를 `useRef`로 관리하여 리스너 등록을 한 번만 수행하도록 조정.
- `setIsVisible`에 함수형 업데이트를 적용하여 방향 판단을 동일하게 유지하면서도 불필요한 상태 갱신을 방지.
- 데스크톱 구간에서는 항상 보이도록 기존 동작을 유지.

## 확인 사항
- 리스너가 한 번만 등록되는지 직접 브라우저에서 스크롤 테스트 필요.
- 모바일/데스크톱 전환 시 메뉴 표시 상태가 기존과 동일하게 유지되는지 확인 권장.
