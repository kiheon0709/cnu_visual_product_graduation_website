## 이미지 파일 네이밍 규칙

### 규칙
1. **소문자 사용**: 파일명은 소문자로 작성
2. **하이픈(-) 사용**: 단어 구분은 하이픈 사용 (언더스코어 대신)
3. **번호 형식**: `-01`, `-02` 형식으로 순차 번호 부여
4. **확장자**: `.jpg` (일반 이미지), `.png` (투명 배경 필요 시), `.svg` (벡터 이미지)

### 예시
- ✅ `home-poster.jpg`
- ✅ `designer-01.jpg`
- ✅ `project-01-thumbnail.jpg`
- ❌ `HomePoster.jpg` (대문자 사용)
- ❌ `designer_01.jpg` (언더스코어 사용)

## 이미지 교체 가이드

### 1. 즉시 사용 가능한 이미지
다음 이미지들은 현재 바로 사용할 수 있습니다:
- 충남대 로고 (`CNU_logo.png`) → `/public/images/logo/cnu-logo.png`로 이동 필요

### 2. 더미 이미지 사용 중인 항목
다음 이미지들은 현재 더미 이미지나 플레이스홀더를 사용 중이며, 실제 이미지 수신 시 교체 필요:
- 메인 포스터 (홈화면사진01)
- 페이지 헤더 이미지들 (포스터가로비율01)
- 디자이너 프로필 이미지 (대형프로필01~, 소형프로필01~)
- 프로젝트 대표 이미지 (프로젝트대표이미지01~)
- 아카이브 포스터 (2023졸업전시 포스터, 2024졸업전시 포스터)

### 3. 이미지 교체 방법

#### 메인 포스터 교체
1. `/public/images/main/home-poster.jpg`에 이미지 추가
2. `app/page.tsx`에서 Image 컴포넌트 주석 해제

#### 프로필 이미지 교체
1. `/public/images/designers/profile-large/` 또는 `profile-small/` 폴더에 이미지 추가
2. 데이터 파일 또는 컴포넌트에서 경로 참조

#### 프로젝트 이미지 교체
1. `/public/images/projects/thumbnails/`에 썸네일 추가
2. `/public/images/projects/gallery/{project-slug}/`에 갤러리 이미지 추가
3. 데이터 파일에서 경로 참조

## 참고사항

### 이미지 최적화
- Next.js Image 컴포넌트 사용 권장
- 자동 WebP/AVIF 변환
- Lazy loading 적용
- `priority` 속성: Above-the-fold 이미지에만 사용

### 이미지 형식 권장사항
- **포스터/배경 이미지**: JPG (큰 용량, 고품질)
- **로고/아이콘**: PNG 또는 SVG (투명 배경 필요 시)
- **프로필 이미지**: JPG 또는 PNG
- **썸네일**: JPG (최적화된 크기)

### 이미지 크기 최적화
- 페이지 로드 속도를 위해 이미지 크기 최적화 권장
- 필요 시 이미지 압축 도구 사용 (예: TinyPNG, ImageOptim)
